## 三層構造

iworの価値は三つの層で構成される。順番が重要。

### 第一層: 速度

3タップで答えが出る。考える前に手が動いている。
当直中に10秒で使えなければ、存在しないのと同じ。

### 第二層: 感情

画面を開いた瞬間の安心感。疲弊した脳に負荷をかけない視覚設計。
「このアプリがあるから大丈夫」と思える信頼感。

### 第三層: 時間の返却

使った結果として、30分早く帰れる。1時間早く寝られる。
それは翌日の患者安全にも直結する。

## 競合が売っているもの、iworが売っているもの

| サービス | 売っているもの | ユーザーにとって |
|---------|-------------|--------------| 
| m3 | 医師のアテンション（→製薬に転売） | ユーザーは商品 |
| HOKUTO | 臨床ツールの「機能」 | 便利だけど、別に好きではない |
| UpToDate | 医学知識の「権威性」 | 信頼できるが、現場の速度に合わない |
| **iwor** | **キャリアと学習が気持ちよく回る「体験」** | **触るたびに安心する。成長が見える。** |

競合はすべて「情報・知識・機能」のレイヤーで戦っている。
**「それを使う人間が、どんな身体的・感情的状態にあるか」を設計に織り込んでいるプレイヤーは、医療ITにまだいない。**

## 他業界のアナロジー

### Linear（vs Jira）— 最も近いアナロジー

LinearはJiraと同じ機能を、圧倒的に気持ちいい体験で提供。
**iwor ↔ Linear**: HOKUTOがJiraなら、iworはLinear。

### Superhuman（vs Gmail）— 価格戦略の参考

Gmailが無料の世界で月$30を取るメールクライアント。
**iwor ↔ Superhuman**: 無料のHOKUTOがある世界で、体験に対価を払う層は存在する。

### Things 3（vs Todoist, Asana）— 個人開発の理想形

「触っているだけで気持ちいい」という理由で熱狂的なファン。買い切り。広告なし。
**iwor ↔ Things 3**: 大きくならなくても強い。愛着が持続するプロダクト。

## 共通パターン: なぜ体験設計で勝てるのか

1. 既存の巨人が「機能・情報・網羅性」で戦っている
2. 後発が「同じことを、圧倒的に気持ちいい体験で」切り込む
3. 機能は同等か、むしろ少ない
4. でも触った瞬間に「あ、違う」と思わせる
5. その「違い」に対価を払う層が、ニッチだけど確実に存在する

**医療はこのパターンが最も効く業界。**

| 新機能を追加するか？ | 医師の体験を速く・気持ちよくするか？ |

| 機能を削るか？ | 削った方が速くなるなら、削る |
| 法務リスクがあるか？ | リスクがあるなら、どんなに便利でも削る |
