## コアバリュー（一文）

> **「臨床体験のデザイン」**
> — 正しい情報を、正しい瞬間に、正しい手触りで届ける。

## 三層構造

iworの価値は三つの層で構成される。順番が重要。

### 第一層: 速度

3タップで答えが出る。考える前に手が動いている。
当直中に10秒で使えなければ、存在しないのと同じ。

### 第二層: 感情

画面を開いた瞬間の安心感。疲弊した脳に負荷をかけない視覚設計。
「このアプリがあるから大丈夫」と思える信頼感。

### 第三層: 時間の返却

使った結果として、30分早く帰れる。1時間早く寝られる。
それは翌日の患者安全にも直結する。

## 競合が売っているもの、iworが売っているもの

| サービス | 売っているもの | ユーザーにとって |
|---------|-------------|--------------|
| m3 | 医師のアテンション（→製薬に転売） | ユーザーは商品 |
| HOKUTO | 臨床ツールの「機能」 | 便利だけど、別に好きではない |
| UpToDate | 医学知識の「権威性」 | 信頼できるが、現場の速度に合わない |
| 個人のnote/PDF | 作った人の「経験知」 | 一度読んで棚にしまう |
| **iwor** | **臨床が気持ちよく回る「体験」** | **触るたびに安心する。早く帰れる。** |

競合はすべて「情報・知識・機能」のレイヤーで戦っている。
**「それを使う人間が、どんな身体的・感情的状態にあるか」を設計に織り込んでいるプレイヤーは、医療ITにまだいない。**

## 他業界のアナロジー

iworと同じポジションで、既存の巨人を崩した（崩しつつある）会社がある。
いずれも「同じことを、圧倒的に気持ちいい体験で」という一点で切り込んでいる。

### Linear（vs Jira）— 最も近いアナロジー

プロジェクト管理ツール。Jiraは機能が膨大だがUIが重く、設定に専任管理者が必要。
Linearはローカルファーストでミリ秒単位の応答速度。キーボードファーストで、マウスに手を伸ばす前に作業が終わる。
Linearを使うチームは熱烈に推薦する。Jiraを使うチームは「まあ、悪くないんじゃない」。

**iwor ↔ Linear**: HOKUTOがJiraなら、iworはLinear。

### Superhuman（vs Gmail）— 価格戦略の参考

Gmailが無料の世界で月$30を取るメールクライアント。100ミリ秒以下で開く。
機能はGmailと同等。でも「速さと気持ちよさ」に月$30払う人がいる。
創業者Rahul Vohra曰く「速度は機能ではなく感情である」。

**iwor ↔ Superhuman**: 無料のHOKUTOがある世界で、体験に対価を払う層は確実に存在する。

### Things 3（vs Todoist, Asana）— 個人開発の理想形

ドイツCultured Code製のTodoアプリ。Todoアプリは世界に何千とあるが、
「触っているだけで気持ちいい」という理由で熱狂的なファンがいる。
買い切り。広告なし。機能追加は極めて慎重。会社は小さいまま。でも持続的に売れ続ける。

**iwor ↔ Things 3**: 大きくならなくても強い。愛着が持続するプロダクト。

### Stripe（vs PayPal）— 体験設計でコモディティ市場を制した例

決済という最も退屈な領域で、「開発者が触って気持ちいいAPI」と「美しいドキュメント」で勝った。
同じ機能を、圧倒的に良い体験で包んだだけで時価総額数兆円の企業になった。

### Figma（vs Adobe）— 体験でエコシステムを崩した例

「ブラウザで開いた瞬間に全員が同じ画面を見られる」という体験で、
Adobe XDやSketchを置き去りにした。Adobeが$200億で買収しようとした事実が価値の証明。

## 共通パターン: なぜ体験設計で勝てるのか

これらの会社の共通構造:

1. 既存の巨人が「機能・情報・網羅性」で戦っている
2. 後発が「同じことを、圧倒的に気持ちいい体験で」という一点で切り込む
3. 機能は同等か、むしろ少ない
4. でも触った瞬間に「あ、違う」と思わせる
5. その「違い」に対価を払う層が、ニッチだけど確実に存在する

**医療はこのパターンが最も効く業界。**
当直中の疲弊、患者急変の緊張、時間に追われるストレス —
体験設計の価値が最も高くなる極限状況が、日常的に発生する。

## 創業者の原体験

- 病院の同期に臨床テンプレート・ショートカットを作って共有するのが好きだった
- 趣味の音楽制作で、DAWミックス・マスタリングテンプレートを販売し、大学生時代に数百万円の売上
- 売っていたのは「知識」ではなく「面倒を消して、本来やりたいことに集中できる時間」

DAWテンプレートの成功とiworは同じ構造:
- 音楽家にとっての「音楽に集中できる時間」
- 医師にとっての「患者に集中できる時間」と「早く帰れる時間」

## 判断基準: すべてここに帰着する

| 判断 | 問い |
|------|-----|
| 新機能を追加するか？ | 医師の体験を速く・気持ちよくするか？ |
| 広告を入れるか？ | 体験を壊すか？ → 壊すなら入れない |
| UIを変更するか？ | 当直中の疲弊した脳に、より優しいか？ |
| 価格を変えるか？ | 体験の価値に見合っているか？ |
| 機能を削るか？ | 削った方が速くなるなら、削る |
| 製薬と連携するか？ | UIに一切出ない形で実現できるか？ |

答えがNoなら、どんなに正しい情報でも、どんなに要望が多くても、iworには入れない。
