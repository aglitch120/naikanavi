## ビジネスモデル

### フリーミアム戦略: PLG（Product-Led Growth）

**原則: LPを読ませるのではなく、体験しながらPROの価値に気づかせる。**

全ツールのUI/操作は完全公開。入力も計算もフロー操作もすべてFREE。
ユーザーが「使ってみたら便利だった→もっと使いたい」体験を経てからPRO訴求。

**安全性ファースト**: 計算結果・ER/ICUの判断フロー・緊急対応は絶対にゲートしない。患者安全に関わる情報の有料化は信頼を破壊する。

### 無料（全ツールのUI/操作 + SEO集客装置）
- 臨床計算ツール（入力＋計算＋結果表示）
- ER対応ツリー（全ステップ＋全分岐＋緊急対応指示）— 完全公開
- ICU管理ツール（全公開）— 緊急性あり
- ACLS・緊急対応フロー（全公開）— 緊急性あり
- 検査読影インタラクティブ（フロー操作＋途中判定）
- 生活習慣病ツール（入力＋判定＋リスク分類）
- 薬剤比較（全種）
- SEOブログ（全記事）
- 論文フィード（最新3件）
- 専門科診断（20問クリック式）
- J-OSLER/病棟ダッシュボード（UIプレビュー）

### PRO限定（モザイク or ロック）
- **解釈セクション**: 計算ツールの「この結果の意味」「推奨アクション」（モザイク表示）
- **検査読影の総合解釈**: 最終結果の鑑別疾患リスト（モザイク表示）
- **生活習慣病アクションプラン**: 具体的な治療方針一覧（モザイク表示）
- **お気に入り保存**: ツールのピン留め
- **データ永続化**: J-OSLER症例、病棟TODO、学習進捗、論文ブックマーク
- **論文フィード全アーカイブ**: 最新3件以外
- **専門科診断結果**: 詳細レポート＋おすすめ科
- **将来追加されるPRO機能すべて**

### PLGタッチポイント（インタラクティブPRO発見）

ユーザーが操作する中で自然にPROに気づく仕組み:
- 初回ツール利用: お気に入りボタンのパルスアニメーション（1回のみ）

## 確定事項・却下仮説

> ※ 以下は戦略議論（2026年3月16日）の結論。同じ議論を繰り返さないための記録。

### 確定: ユーザーライフサイクル

```
医学5-6年生 → マッチング対策で入会（毎年9,000人のTAM）
  ↓ そのまま継続
初期研修医 → stat tracker + TODOで継続（経験症例の分布蓄積）
  ↓ そのまま継続
専攻医 → J-OSLER進捗 + stat trackerで継続
```

1人のLTV = ¥9,800 × 3〜5年 = ¥29,400〜¥49,000

### 確定: ロックイン戦略

患者データには一切触れない。キャリアデータでロックイン:
- マッチングプロフィール + 志望順位 + AI面接履歴
- 病棟stat tracker: 経験疾患の分布（疾患カテゴリ + 件数のみ。患者名なし）
- J-OSLER症例カウント + 進捗数値
- 論文ブックマーク + メモ

### 確定: 病院DB方針

- 公開データの自動集約ページのみ（厚労省REIS + マッチング倍率）
- 口コミ機能は入れない
- マッチングツールのデータ基盤として活用

### 確定: 集合データ資産

プラポリに統計利用条項を入れる。売却時の資産になる匿名集合データ:
- 研修医の経験疾患分布（全国平均との比較）
- マッチング行動データ（志望科・地域・倍率の傾向）
- 論文閲覧傾向、専門科診断の結果分布

### 却下した仮説

| 仮説 | 却下理由 |
|------|---------|
| 患者データでロックイン | 要配慮個人情報。法的リスクが事業を脅かす |
| On-device AI（ブラウザ内LLM） | 3省2ガイドライン未対応。ブラウザの周辺リスク残存。時期尚早 |
| Anki型適応学習 | 医師は試験慣れ。参考書+過去問で十分。年額課金の動機にならない |
| 病院口コミDB | HOKUTO先行（口コミNo.1, 企業価値75億）。修了者の投稿モチベなし |
| B2B2C（機関導入） | 営業力が必要。今の段階では非現実的。将来再検討 |

## 競合分析

### HOKUTOの構造と弱点

**強み:** 会員10万人、計算ツール500+種、製薬広告で完全無料、口コミNo.1、企業価値75億円
**収益モデル:** 製薬企業のマーケティング費用。利用者データの収集・分析で製薬マーケティングを支援

**構造的弱点（= iworのチャンス）:**
1. 臨床ツールがアプリ内閉鎖 → Web検索に出ない → iworがSEO空白地帯を獲る
2. 病院がスポンサー → 中立性に疑問（ただし口コミ量で圧倒しているので実害小）
3. マッチング対策ツール（AI面接等）はない → iworの差別化ポイント
4. 病棟stat tracker / キャリアデータ蓄積機能はない → iworの独自価値

### 競合マップ

| 競合 | 強み | iworの攻め方 |
|------|------|-------------|
| HOKUTO | 10万人, 500+ツール, 無料 | Web公開型でSEO空白地帯を獲る |
| UpToDate | 臨床判断の権威 | 競合しない（共存） |
| m3 | 30万人医師会員 | 競合しない（層が違う） |
| レジナビ | マッチング情報の老舗 | AI面接等で差別化 |
| 医師ナビ | 口コミ+構造化データ | 公開型+マッチング連携 |
