## 4. 散文ファースト（最重要ルール）

**各H2/H3セクションは必ず2〜3段落の散文から始める。H2/H3直後に箇条書きを置くのは禁止。**

各段落を書く前に以下の3つを自問し、最低1つを散文に含める：
1. **なぜ？** この情報がなぜ重要か（背景・理由）
2. **どう問われる？** 読者が実際に直面する場面を描写
3. **何を間違えやすい？** よくある誤解・混同ポイントを先回りして指摘

### NG（AIが書きがちなパターン）

```
## バイトの種類

**健診バイト**
- 午前のみ2〜3万円
- 翌日への影響少
```
→ H2直後に箇条書き。「なぜ健診バイトが向いているか」の説明がゼロ。

### OK（散文ファースト）

```
## バイトの種類

専攻医に最も向いているのは健診バイトです。理由は翌日への体力的影響が少ないためで、
J-OSLERの病歴要約作業と並行しやすいという特徴があります。...

**健診バイト**は問診・身体診察・心電図判読が主な業務で、
内科研修を修了した専攻医であれば即戦力として働けます。...
```

## 5. 太字・文体のルール

### 太字

- 各段落に1〜2箇所の太字を自然に配置（重要な数値・結論・キーワード）
- 箇条書き先頭だけの「テンプレ太字」は禁止
- **太字の中に全角括弧（）を入れてはいけない。括弧は必ず** の外に出す

```
NG: **iDeCo（個人型確定拠出年金）**  → **がそのまま表示される
OK: **iDeCo**（個人型確定拠出年金）   → 括弧を外に出す
```

### 文体

- 「〜です」「〜ます」の丁寧語。教科書的な「〜である」は使わない
- 「〜してください」「〜を覚えておきましょう」など読者への語りかけを含める
- 一文は60文字以内。1段落は4文以内

## 6. SVG図解のルール

### 必須：記事ごとに異なる2〜3種類を使う

| SVGタイプ | 適する場面 |
|-----------|-----------|
| フローチャート | 手順・判断プロセス |
| タイムライン | 時系列の流れ |
| 比較図（左右） | 2つの概念の対比 |
| 優先度マップ | 学習・作業の優先順位 |
| ステップ図（横並び） | 段階的なプロセス |
| カード型レイアウト | 要件一覧・要素の並列 |
| 警告・注意図 | やってはいけないこと |
| 棒グラフ・カレンダー | 量・頻度の視覚化 |

**同じ記事で同じタイプを2つ使うことは禁止。連続する記事で全く同じ構造のSVGを使うことも禁止。**

### 必須要素

```xml
<rect width="800" height="XXX" rx="16" fill="#F5F4F0"/>
<rect x="24" y="20" width="752" height="44" rx="8" fill="#1B4F3A"/>
<text x="400" y="48" text-anchor="middle" font-family="sans-serif"
      font-weight="bold" font-size="16" fill="#fff">タイトル</text>
<defs>
  <filter id="fs" x="-4%" y="-4%" width="108%" height="108%">
    <feDropShadow dx="0" dy="2" stdDeviation="4" flood-color="#000" flood-opacity="0.08"/>
  </filter>
</defs>
<text x="400" y="YYY" text-anchor="middle" font-family="sans-serif"
      font-size="11" fill="#6B6760">© 内科ナビ naikanavi.com</text>
```

### レイアウト注意

- viewBoxの高さ >= 最下部要素 + 40px + フッター高さ(20px)
- テキストが図形の辺と重なる場合、テキスト背景に白rect を配置
- テーブルのカラム数は4以下を推奨（5以上はモバイルではみ出し）

## 8. 内部リンク・タグリンク・CTA の404防止ルール（最重要）

**リンク切れは検索評価を直接毀損する。以下は絶対遵守。**

### 内部リンク（/blog/slug）

```bash
# リンクを書く前に必ず存在確認
ls content/blog/ | grep "スラッグ名"
```

- 存在しない記事へのリンクは絶対に書かない → テキストのみにする
- 「準備中」「近日公開」の記事へのリンクは禁止

### タグのリンク

- タグURLは lib/blog-config.ts の tagSlugMap に登録済みのスラッグのみ有効
- 未登録タグを使う場合は tagSlugMap に追加してからコミットする

### cta_type（フロントマター）

有効値: `template` / `progress` / `quiz` / `checklist` / `general` のみ
それ以外の値を設定しない。

### アンカーテキスト

```
NG: [こちら](/blog/xxx)
NG: [この記事](/blog/xxx)
OK: [J-OSLER病歴要約の書き方完全ガイド](/blog/c01-josler-byoreki-youyaku-kakikata)
```

## 9. コミット前チェックスクリプト（必須実行）

スクリプトを docs/check_quality.sh として保存済み。

```bash
# 使い方（リポジトリルートから実行）
bash docs/check_quality.sh content/blog/ARTICLE_SLUG.mdx
```

**全項目✅になってからコミットする。1項目でも❌があればコミットしない。**

チェック内容：
1. SVG数（2個以上）
2. CTA数（2箇所以上）
3. 内部リンク数（3本以上）
4. よくある失敗セクション
5. 文字数（12000バイト以上）
6. 太字内の全角括弧（0件）
7. 内部リンク404チェック（全スラッグの実在確認）
8. cta_typeの有効値チェック
9. H2直後の箇条書きチェック（散文ファースト違反検出）

## 11. よくある失敗パターンセクション

配置：まとめの直前
形式：パターン1/2/3それぞれ3〜5行の散文で解説

失敗パターンの探し方：
- 数値の混同（120を160と覚える等）
- 手順の逆転（届出なしでバイト開始等）
- 過剰行動（バイト入れすぎてJ-OSLER修了遅延等）
- 申告漏れ・期限忘れ
