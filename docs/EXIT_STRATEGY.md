## 2. プロダクト強化ロードマップ（優先順位順）

### Phase A: 病歴要約の出力品質強化（最優先）

**目標: 「内科ナビで作ったテンプレが一番使える」状態**

#### A-1. テンプレート出力の高度化
- [ ] 文字数カウンター（リアルタイム表示、J-OSLER推奨文字数との対比）
- [ ] 検査値自動挿入テンプレ（疾患ごとの必須検査値一覧を出力に含める）
- [ ] 考察テンプレート強化
  - 全人的医療の記載テンプレ（bio-psycho-social model）
  - 疾患ごとの論点テンプレ（診断根拠・治療選択理由・予後）
  - 倫理的考察テンプレ（IC・ACP・意思決定支援）
- [ ] 推奨参考文献の自動出力（疾患ごとに2-3本のガイドライン・高IF論文）

#### A-2. AI品質のステップアップ
- [ ] Claude API連携によるテンプレ生成（現在の静的テンプレ→AI動的生成）
  - ユーザーが入力した患者情報に基づくカスタマイズ
  - 実際の検査値を入力→適切な考察文を生成
- [ ] ファクトチェック機能（生成内容をガイドラインと照合）
- [ ] 差し戻し対策AI（J-OSLERの差し戻しパターンを学習、事前チェック）

#### A-3. 出力フォーマット
- [ ] コピペ用プレーンテキスト出力
- [ ] Word形式ダウンロード（J-OSLER提出用）
- [ ] 過去の生成履歴からの再編集機能

**技術実装メモ:**
- 現状: worker.jsのTMPLオブジェクト（370疾患×静的テンプレ）
- 改善: Claude API（claude-sonnet-4-20250514）を使い、静的テンプレをベースにAI動的生成
- Cloudflare WorkersからAnthropic APIを呼び出す構成

## 3. SEO自動運営システム

### 3-1. GA4 + GSC連携

- [ ] GA4トラッキング設置（layout.tsxに追加）
- [ ] GSCデータのAPI取得
- [ ] 記事パフォーマンスの自動集計

### 3-2. リライト判断基準

| 条件 | アクション |
|------|----------|
| 公開後3ヶ月で順位11位以下 | リライト候補 |
| CTR < 3%（順位5位以内なのにCTR低い） | タイトル・ディスクリプション改善 |
| 直帰率 > 80% + 滞在時間 < 1分 | コンテンツ構成の見直し |
| 競合が新規記事で上位に | 差分分析→追記 |

### 3-3. 競合監視

**監視対象サイト:**
- おちば先生ブログ
- らーめんどくたーブログ
- 病歴要約アシスト
- かめちゃんブログ

**監視項目:**
- 新規記事の投稿（RSS/手動チェック）
- 検索順位の変動
- 新機能のリリース

### 3-4. 新規記事投稿基準

| 基準 | 頻度 |
|------|------|
| ガイドライン改訂（年1-2回） | 即座にリライト |
| 新規トピック（J-OSLER制度変更等） | 1週間以内に投稿 |
| ロングテール補充 | 月2-4本 |
| 季節性コンテンツ（試験前・新年度） | 該当時期の2ヶ月前 |

### 3-5. 自動リライトシステム（段階的構築）

**Phase 1（今すぐ）:** 手動チェック + リライト判断基準に基づくタスク生成
**Phase 2（GA4連携後）:** ダッシュボードにリライト推奨記事を自動表示
**Phase 3（将来）:** Claude APIで下書き生成 → 人間がレビュー → 公開
