## 3. データ保存: Worker KV

### エンドポイント

#### PUT /api/conferences/attend — 参加予定登録/解除

```
認証: sessionToken必須
Body: { conferenceId: string, action: "add" | "remove" }

処理:
1. authenticate (checkExpiry: false — FREE/PRO両方可)
2. ユーザーの参加リストを取得/更新
   KV: conf-user:{email} → { attending: ["naika-2026", "geka-2026", ...] }
3. FREEユーザーは最大3件チェック → 超過時はProGateモーダル返却
4. 該当学会の参加者カウント更新
   KV: conf-count:{conferenceId} → number

レスポンス: { ok: true, attending: [...], count: number }
```

#### GET /api/conferences/counts — 参加予定者数一括取得

```
認証: 任意（認証で分岐）
Query: ?ids=naika-2026,geka-2026,...

レスポンス（PRO）:
{ counts: { "naika-2026": 42, "geka-2026": 15, ... } }

レスポンス（FREE）:
{ counts: { "naika-2026": "10+", "geka-2026": "10+" }, blurred: true }
※ 実数ではなく「○人以上」でぼかす
```

#### GET /api/conferences/my — 自分の参加予定リスト

```
認証: sessionToken必須
レスポンス: { attending: ["naika-2026", "geka-2026", ...] }
```

### KVキー設計

```
conf-user:{email}      → { attending: string[] }（ユーザーの参加予定リスト）
conf-count:{confId}    → number（学会ごとの参加予定者数）
```
