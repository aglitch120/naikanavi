「ChatGPTに文献を聞いたら存在しない論文を出された...」「PubMedで英語検索するのがそもそもしんどい...」

病歴要約の引用文献探しや学会発表の文献レビューで、AIに頼りたい場面は多いですよね。でも従来のAIは **学習データから「それっぽい」論文情報を生成してしまう** 問題（ハルシネーション）があり、引用には使いにくいのが実情でした。

この問題を解決するのが **Claude + PubMed MCP連携** です。MCPという仕組みを使うと、ClaudeがPubMedのデータベースに **直接アクセス** して、実在する論文だけを返してくれます。この記事では「MCPって何？」というところから、実際の設定方法、病歴要約での活用例まで解説します。

> [内科ナビ](https://naikanavi.booth.pm/items/8058590)のAI病歴要約テンプレートは、Claude PubMed MCPでPubMedの実データと照合済みの引用文献候補を提示します。全70疾患群・373疾患対応。

## そもそもMCPとは？｜医師向けにざっくり説明

**MCP（Model Context Protocol）** はAnthropic社（Claudeの開発元）が作った、AIと外部サービスをつなぐための規格です。

これだけだとわかりにくいので、たとえ話で説明します。

普通のAI（ChatGPTやClaude単体）は、いわば **「教科書を丸暗記した研修医」** です。質問すると記憶から答えてくれますが、記憶にない情報は「うろ覚えで」答えてしまうことがあります。存在しない論文を自信満々に教えてくれるのがまさにこれです。

MCP連携したClaudeは、 **「電子カルテや文献検索端末にアクセスできる研修医」** です。質問されたら、まず端末（PubMed）で調べてから答えてくれます。だから回答が正確になるのです。

![Claude + PubMed MCP の仕組み](/images/blog/j10-claude-pubmed-mcp-shikumi.svg)

ポイントは、Claudeが **PubMedの実際のデータベースに直接クエリを投げる** 点です。AIの記憶から論文を「思い出す」のではなく、PubMedのAPIを通じてリアルタイムに検索結果を取得します。だから返ってくるのは必ず実在する論文であり、PMID・著者名・掲載誌もすべて正確です。

## なぜ医師にとって重要なのか？

### ハルシネーション（論文の捏造）を防げる

AIのハルシネーションとは、AIが事実と異なる情報をもっともらしく生成してしまう現象です。ChatGPTやClaude（MCP未使用時）に「心不全のランドマーク試験を教えて」と聞くと、だいたいは正しい論文を返してくれますが、たまに **存在しない論文** や **著者名が間違った論文** を混ぜてくることがあります。

病歴要約の引用文献で存在しない論文を書いてしまうと、査読で確実に指摘されます。PubMed MCPを使えば、このリスクをほぼゼロにできます。ちなみに[内科ナビ](https://naikanavi.booth.pm/items/8058590)のAI病歴要約テンプレートは、このClaude PubMed MCPで実データと照合済みの引用文献候補をあらかじめ提示しているので、自分でMCPを設定しなくても正確な文献候補が手に入ります。

### 英語が苦手でもPubMed検索ができる

PubMedの検索画面を直接使う場合、MeSH用語やBoolean演算子（AND/OR/NOT）を英語で組み立てる必要があります。MCP連携なら **「心不全のGDMTに関するRCTを5本探して」** と日本語で指示するだけで、Claudeが適切な検索クエリを自動生成してPubMedから論文を取得してくれます。

### 論文情報を引用形式に自動変換できる

取得した論文のPMID・著者・タイトル・掲載誌・年を、病歴要約の引用形式（Vancouver形式など）に自動で整形するよう指示できます。手作業でPubMedからコピペして書式を整える手間がなくなります。

## PubMed MCPの設定方法

PubMed MCPの導入にはいくつかの方法があります。一番手軽な方法から紹介します。

![PubMed MCP 導入3ステップ](/images/blog/j10-claude-pubmed-mcp-setup.svg)

### 方法1: claude.ai のインテグレーション（最も簡単）

**claude.ai**（ブラウザ版）の有料プラン（Pro以上）を使っている場合、設定画面の「インテグレーション」からPubMed MCPを有効にするだけで使えます。デスクトップアプリのインストールやJSON設定ファイルの編集は不要です。

設定手順は以下のとおりです。

- claude.aiにログイン
- 左メニューの設定アイコンをクリック
- 「インテグレーション」を選択
- 利用可能なMCPサーバー一覧から「PubMed」を探して有効化
- 設定完了。チャット内で「PubMedで心不全のRCTを検索して」と入力すれば動作する

### 方法2: Claude Desktopアプリ + 設定ファイル

Claude Desktopアプリ（Mac/Windows）を使う場合は、設定ファイルにPubMed MCPサーバーの情報を追記します。

まず準備として、Claude Desktop（[公式ダウンロード](https://claude.ai/download)）とNode.js（[公式サイト](https://nodejs.org)からLTS版）をインストールしてください。どちらも無料です。

次に、Claude Desktopの設定ファイルを編集します。ファイルの場所はOSによって異なります。

- **Mac**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

Claude Desktopのメニューから「ファイル → 設定 → 開発者 → 構成を編集」でもこのファイルを開けます。

設定ファイルにPubMed MCPサーバーの情報を追記し、Claude Desktopを再起動すれば完了です。チャット画面にハンマーアイコンが表示されれば、MCPが有効になっています。

具体的なJSON設定の書き方は、使用するPubMed MCPサーバーの公式ドキュメントに従ってください。サーバーは複数公開されており、それぞれ設定方法が異なります。

### 方法3: Claude Code（エンジニア向け）

ターミナルで動くClaude Codeを使う場合は、`claude mcp add` コマンドでMCPサーバーを追加できます。これはプログラミングに慣れた方向けの方法です。

## 実践: 病歴要約の引用文献をClaude + PubMed MCPで探す

設定が終わったら、実際に使ってみましょう。病歴要約の文献検索を例に、具体的な使い方を紹介します。

### ステップ1: 疾患と検索テーマを伝える

Claudeに以下のように入力します。

```
PubMedで検索してください。
テーマ: HFrEF（収縮能低下型心不全）における薬物治療のエビデンス
条件: RCTまたはメタアナリシス、2018年以降、英語論文
希望本数: 5本程度
```

Claudeは自動的にPubMedの検索クエリを組み立て、条件に合う論文を取得してくれます。

### ステップ2: 結果を確認する

Claudeは論文ごとに以下の情報を返してくれます。

- **タイトル**（英語）
- **著者**（筆頭著者＋et al.）
- **掲載誌・年・巻・ページ**
- **PMID**（PubMedの識別番号）
- **概要**（日本語で要約してくれる）

PMIDがあるので、`https://pubmed.ncbi.nlm.nih.gov/PMID番号/` にアクセスすれば原著論文を確認できます。

### ステップ3: 引用形式に変換する

病歴要約で使う引用形式に変換するよう指示します。

```
上記の論文を以下の形式で出力してください。
形式: 著者名. タイトル. 掲載誌. 年;巻:ページ.
```

手入力で書式を整える手間がなくなり、転記ミスも防げます。

### 注意点: 最終確認は必ず自分で

PubMed MCPを使えばハルシネーションのリスクは大幅に下がりますが、ゼロにはなりません。検索結果から論文を選ぶのはClaudeの判断であり、テーマと関連性の低い論文が混ざる可能性はあります。提出前に必ず **PMIDリンクから原著論文のAbstractを確認** してください。

> 文献探しに時間をかけたくないなら、[内科ナビ](https://naikanavi.booth.pm/items/8058590)のAI病歴要約テンプレートが便利です。疾患群と疾患名を選ぶだけで、考察の方向性と引用文献候補（PubMed実データ照合済み）まで骨組みが手に入ります。

## ChatGPT・Perplexityとの比較

文献検索に使えるAIツールを比較します。

### ChatGPT

ChatGPTには「Browse with Bing」機能がありますが、これはPubMedに特化した検索ではなく、Web全体を検索するものです。PubMedに直接アクセスする仕組みではないため、論文情報の正確性はClaude + PubMed MCPに劣ります。ただし有料版（Plus / Pro）で利用できるGPT-4は推論能力が高く、文献の内容理解には優れています。

### Perplexity

Perplexityはリアルタイム検索が強みで、出典のURLを明示してくれます。PubMedの論文もWeb検索経由でヒットすることがありますが、検索クエリのカスタマイズ性（MeSH用語の指定、発行年の絞り込みなど）はPubMed MCP連携に及びません。最新のニュースやガイドラインの更新情報を素早くキャッチするのには向いています。

### Claude + PubMed MCP

PubMedのデータベースに **直接** アクセスするため、論文メタデータ（PMID、著者、掲載誌）の正確性が最も高いです。MeSH用語や発行年での絞り込みなど、PubMedの検索機能をフルに活用できます。デメリットは初回の設定が必要な点と、Claudeの有料プランが必要な点です。

病歴要約の引用文献検索には **Claude + PubMed MCP** が最適解ですが、ガイドラインの最新版チェックにはPerplexityを、英語論文の内容理解にはChatGPTをと、用途に応じて使い分けるのがおすすめです。「MCPの設定が面倒」「有料プランに入るほどではない」という方は、[内科ナビ](https://naikanavi.booth.pm/items/8058590)のAI病歴要約テンプレートが現実的な選択肢です。疾患群と疾患名を選ぶだけで、PubMed実データ照合済みの引用文献候補を含む病歴要約の骨組みが手に入ります。自分でClaudeの設定をする必要はありません。AI活用全般の方法は[J-OSLERをAIで効率化する方法](/blog/j01-josler-ai-katsuyo)で詳しく解説しています。

## 活用シーン: 病歴要約以外にも

PubMed MCP連携は病歴要約の文献検索だけでなく、さまざまな場面で役立ちます。

**学会発表・論文執筆**: 文献レビューの初期段階で関連論文を網羅的に収集し、研究の位置づけを把握するのに最適です。「この疾患のシステマティックレビューを探して」「2020年以降のRCTに絞って」といった指定が自然言語でできます。

**日常臨床の疑問解決**: 「この薬剤の腎機能低下時の用量調整に関するエビデンスは？」「高齢者のDOAC選択に関する比較試験は？」といった臨床的な疑問に、エビデンスベースで答えを探せます。

**内科専門医試験対策**: ガイドラインの根拠となった大規模臨床試験（PARADIGM-HF、DAPA-HF、EMPEROR-Reducedなど）の原著論文を手軽に確認できます。苦手分野の知識を固めたいなら、[内科ナビ](https://naikanavi.booth.pm/items/8058590)の問題演習機能（疾患ごとの5択問題、苦手分野の自動判定付き）もあわせて活用すると効率的です。試験対策全般については[内科専門医試験の勉強法](/blog/g01-naika-senmoni-benkyouhou)も参考にしてください。

> 病歴要約の基本的な書き方から確認したい方は[病歴要約の書き方完全ガイド](/blog/c01-josler-byoreki-youyaku-kakikata)を、文献引用の形式については[文献引用ルール](/blog/c07-josler-bunken-inyo)をご覧ください。

## まとめ

Claude + PubMed MCP連携のポイントを整理します。

- **MCP** はAIと外部サービスをつなぐ規格。PubMed MCPを使うとClaudeがPubMedに直接アクセスして論文を検索する
- 従来のAI文献検索で問題だった **ハルシネーション（存在しない論文の生成）** のリスクを大幅に低減できる
- 日本語で指示するだけでPubMed検索ができ、英語の検索クエリを組み立てる必要がない
- 設定は **claude.aiのインテグレーション設定** が最も簡単（有料プランが必要）
- 病歴要約の引用文献検索、学会発表の文献レビュー、日常臨床の疑問解決に幅広く活用できる
- 最終確認は必ず自分でPMIDリンクから原著論文のAbstractを確認すること
- 自分でMCPを設定するのが難しければ、[内科ナビ](https://naikanavi.booth.pm/items/8058590)のAI病歴要約テンプレート（PubMed実データ照合済み引用文献候補付き）を活用するのが手軽

## よくある質問

### MCPの利用にお金はかかりますか？

MCP自体はオープンソースの規格であり無料です。ただし、Claudeの有料プラン（Pro: 月額約20ドル）が必要です。PubMedのAPI利用自体は無料ですが、大量のリクエストを送る場合はNCBIのAPIキー取得が推奨されています（取得も無料）。

### PubMed以外のデータベース（医中誌、Cochraneなど）にもMCPで接続できますか？

現時点ではPubMedのMCPサーバーが最も充実しています。医中誌やCochrane Library向けの公式MCPサーバーはまだ公開されていませんが、MCPは汎用的な規格のため、今後対応が広がる可能性があります。Cochrane Reviewsの多くはPubMedにも収載されているため、PubMed MCPで検索可能です。

### AIの生成した引用文献をそのまま病歴要約に使っていいですか？

日本内科学会は2024年11月に生成AI利用ガイドラインを公表し、条件つきでAI活用を認めています。ただし「AIが出力した内容の正確性は専攻医自身が確認すること」が原則です。PubMed MCPで取得した論文であっても、必ずPMIDリンクから原著を確認し、自分の症例との関連性を自分で判断した上で引用してください。AI活用の詳しいルールは[J-OSLERをAIで効率化する方法](/blog/j01-josler-ai-katsuyo)を参照してください。

### スマホでも使えますか？

Claude Desktopアプリはパソコン（Mac/Windows）専用です。スマホのClaude公式アプリからもMCPを利用できる場合がありますが、機能が制限されることがあります。文献検索はパソコンで行い、結果をメモアプリ等に保存して持ち歩くのが実用的です。
