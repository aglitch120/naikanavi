# 内科ナビ 改善ロードマップ v1.0

> 作成日: 2026年3月9日
> 目的: naikanavi.com をSEO・デザイン・ユーザビリティ・拡張性の観点から体系的に改善する

---

## 現状分析サマリー

### 強み
- Next.js + MDX の基盤は構築済み
- デザインシステム・SEOガイドラインが文書化されている
- 構造化データ（JSON-LD）の実装あり
- トピッククラスター戦略が設計済み
- robots.ts / sitemap.ts 実装済み

### 課題
1. **コンテンツ不足**: 記事が1本のみ（目標170本中）
2. **Googleインデックス未確認**: 検索で naikanavi.com がヒットしない
3. **技術的SEO不備**: MDXのHTML変換が未実装（dangerouslySetInnerHTML に生markdownを渡している）
4. **ページ不足**: /privacy, /terms, /contact が未作成
5. **パフォーマンス**: next/image 未使用、Webフォント未最適化
6. **GEO（AI検索）未対応**: FAQ構造化データ、HowTo構造化データなし
7. **分析ツール未接続**: GA4, GSC 未設定の可能性
8. **競合が先行**: おちば先生・らーめんどくたー・病歴要約アシストが上位

---

## フレームワーク: 4段階改善モデル

```
Phase 0: 基盤修正（即座に着手）    ← 技術的な問題をまず解消
Phase 1: SEO基盤強化（1-2週）      ← 検索エンジンに正しく認識される状態へ
Phase 2: コンテンツ戦略実行（1-3月）← 記事を体系的に投入
Phase 3: UX・拡張性（2-6月）       ← ユーザー体験と長期的な成長基盤
```

## Phase 0: 基盤修正（即座に着手）

### 0-1. MDX→HTML変換の修正 [優先度: 最高] ✅ 完了 (2026-03-09)
- **問題**: `[slug]/page.tsx` が `dangerouslySetInnerHTML={{ __html: content }}` で生markdownを出力
- **対策**: MDXコンテンツを適切にレンダリングするパイプライン構築
  - `next-mdx-remote/rsc` を導入し `compileMDXContent()` 関数を実装
  - rehype-raw, rehype-highlight など追加
- **工数**: 2-3時間

### 0-2. 法的ページの作成 [優先度: 高] ✅ 完了 (2026-03-09)
- `/privacy` - プライバシーポリシー
- `/terms` - 利用規約
- `/contact` - お問い合わせ（Googleフォーム埋め込みでOK）
- `/tokushoho` - 特定商取引法に基づく表示（BOOTH販売なら必須）
- **工数**: 各30分

### 0-3. Google Search Console 登録・確認 [優先度: 最高]
- GSCにnaikanavi.comを追加
- サイトマップ送信 (`/sitemap.xml`)
- インデックス登録リクエスト
- **工数**: 30分

### 0-4. Google Analytics 4 設置 [優先度: 高]
- GA4トラッキングコードをlayout.tsxに追加
- イベント設定（BOOTH遷移、アプリログイン、CTA クリック）
- **工数**: 1時間

|------|---------|------------|

| ★★☆ | 試験対策 | 勉強法、おすすめ参考書、合格率・難易度 |

#### Tier 3: ロングテール記事（145本）
残りの記事を3ヶ月かけて体系的に投入。

### 2-2. コンテンツ品質チェックフロー

```
1. キーワード選定（検索ボリューム・競合度確認）
2. 競合上位3記事の構成分析
3. アウトライン作成（H2/H3構造）
4. 本文執筆（SEOガイドライン準拠）
5. 医学的正確性チェック
6. 内部リンク3本以上設置
7. CTA配置（上部・中盤・下部）
8. メタデータ設定
9. 公開 → GSCでインデックスリクエスト
10. 1週間後に順位確認
```

### 2-3. GEO（生成AI検索）対策
- 記事内にFAQセクションを必ず設置
- 定義・数値・手順を明確に構造化
- 「〜とは」形式の見出しを積極使用
- 箇条書きと表で情報を構造化
