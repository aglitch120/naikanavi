## 新コアプロダクト: iwor Study

### 概要

FSRS（最新の間隔反復アルゴリズム）搭載のフラッシュカードアプリ。日本の医学教育に特化。

### 海外リサーチ

- Anki: 米国医学生の86.2%が使用、UIが2010年代のまま
- AnkiHub: 月$5、10万人以上。コミュニティデッキのリアルタイム更新が課金動機
- AMBOSS: 年$428、170カ国200万人。Qbank+Library+Anki統合
- 日本に存在しないもの: AnkiHub的コミュニティデッキ管理、AMBOSS的統合学習

### フリーミアム設計（AnkiHub/Brainscapeモデル準拠）

**FREE（永久無料）— 「一人で学ぶ」は無制限:**
- カード作成・編集・FSRS間隔反復・自作デッキ数・1日レビュー枚数: すべて無制限
- デフォルトデッキ3つ（CBT基礎/国試必修/内科基礎）
- .apkgインポート
- 基本統計（今日の枚数、ストリーク）

**PRO（月額¥980 / 年額¥9,800）— 「他人のデータ・AI・利便性」に課金:**
- コミュニティデッキ購読+リアルタイム更新同期
- AIカード生成（PDF/ノート→カード自動変換）
- 詳細統計・弱点分析・予測スコア
- **ソーシャルプルーフ**: 人気デッキランキング + 「○人が学習中」+ デッキ別平均正答率
- ゲーミフィケーション（XP/リーグ）
- **ストリーク全国ランキング**: 連続学習日数の全ユーザーランキング（PRO限定閲覧）+ 自分の順位表示。医学生のプライド・承認欲求を刺激し、アカウント育成動機を作る
- ストリーク凍結権（1日フリーズ、PRO限定）— ランキング維持のためにPRO解約できなくなる
- SNSシェア機能（Instagramストーリーズ等）
- 試験パーソナライズドスケジュール
- 奨学金制度（払えない人向け — AnkiHub同様）

### リスク評価

- 確信度70%: 日本の医学生にAnki的需要はあるが、QBカルチャーとの競合あり
- コミュニティデッキ自走の確信度: 40%
- 先行者利益: 50%
- **対策: 全ベットではなく段階的検証（実装ロードマップ参照）**

### データロックイン + 損失回避カウンター

PROの真の価値は「使うほど溜まるパーソナルデータ」。退会＝全データ消失（30日猶予期間後）。
- Study学習履歴 / 研修記録 / J-OSLER症例 / マッチングプロフィール / 論文ブックマーク / お気に入り

**損失回避カウンター（解約抑止）:**
- PROモーダル・解約画面に蓄積データを数字で表示
- 例: 「あなたのデータ: Study 847枚の学習履歴 / J-OSLER 23症例 / 論文ブックマーク 15件」
- 「失うもの」の可視化で解約心理ハードルを上げる（Spotify/Notion同様のパターン）

### ファーストテイスト無料モデル（7日間PRO体験の代替）

期間限定の全機能解放は「一度見たら終わり」のソーシャルプルーフ系機能と相性が悪い。代わりに各機能で「初回1回だけ無料で見せる」方式を採用。

| サービス | 無料テイスト | PRO（2回目以降） |
|---------|------------|----------------|
| J-OSLER/EPOC | 初回ベンチマーク1回無料 | 毎月更新されるデータは全てPRO |
| マッチング | 人気TOP3だけ無料表示 | 全順位+気になる数はPRO |
| Study | 人気デッキ1位だけ無料 | 全ランキング+学習者数+正答率はPRO |
| 学会カレンダー | 「○人以上が参加予定」とぼかし | 正確な数字はPRO |

初回hookで「もっと見たい」→ 継続的なPRO転換動機を生む。

### 週間ダイジェスト通知（Variable Reward）

毎週月曜にメール/プッシュ通知。内容が毎回変わる（Variable Reward）から開封率が高い。
- 「先週のiwor: Studyで○枚復習 / 新着人気デッキ / 同期平均との差」
- FREE/PRO問わず全員に送信。ただしPRO限定の数値は「???」でぼかす
- 離脱しかけたユーザーを引き戻すリテンション装置

### ローンチブースト戦略（シードデータ + シードユーザー生成）

ソーシャルプルーフは「最初の利用者がゼロ」だと機能しない。黎明期の対策:

**架空アクティブユーザー100人の生成（ローンチ前に実施）:**
- スクリプトでKV/DBに100アカウント分のデータを投入
- 名前: 匿名表示（「医学生A」「研修医32」等）なので架空でも違和感なし
- セグメント分布: 医学生50人、初期研修医30人、専攻医20人（実際のターゲット比率）
- Studyストリーク: 1〜180日のリアルな分布（正規分布、中央値30日）
- デッキいいね: デフォルト3デッキにそれぞれ20〜50いいね
- マネーランキング: 各カテゴリ10〜30 upvote
- マッチング「気になる」: 45病院にリアルな偏り（都市部の有名病院に集中）
- J-OSLER進捗: 専攻医20人分のベンチマークデータ（0〜70疾患群の分布）
- 学会「参加予定」: 主要学会に5〜15人ずつ

**シードデータの原則:**
- 表示は全て匿名（個人特定不可）なので、架空データでも倫理的問題なし
- 実ユーザーが50人超えた段階で、架空データを段階的にフェードアウト（実データ比率が上がれば自然に薄まる）
- 「○人が利用中」の総数表示は実ユーザー数のみ。架空ユーザーは集計データ（平均・分布・ランキング）にのみ寄与

**その他のシードデータ:**
- ランキングの初期データ: 運営側の実体験ベースで投入
- デフォルトデッキに「いいね」初期値: 運営+シードユーザーで最低限の数字を作る
- 目標: 実シードユーザー30人 + 架空100人 = ソーシャルプルーフ機能の初日から有効化可能

## 自律運用システム（将来ビジョン）

iworを「1つの自走する会社」として運用するAIエージェントシステム。

### アーキテクチャ: 6エージェント + 1オーケストレーター

| エージェント | 頻度 | 入力 | 出力 |
|------------|------|------|------|
| **CEO Orchestrator** | 週1 | 全レポート + docs/ + メトリクス | STRATEGY/TODO更新提案、各エージェントへの指示 |
| **Research** | 日次+週次 | 競合サイト、X/SNS、ユーザーの声 | COMPETITOR_DATABASE更新、機能提案 |
| **Dev** | オンデマンド | CEOの指示、bug報告 | git commit + PR + デプロイ |
| **Marketing** | 日次 | 投稿テンプレ、エゴサ結果、GSC | X投稿ドラフト、SEOレポート |
| **Analytics** | 日次 | GA4、Cloudflare、KV | メトリクスダッシュ、チャーンアラート |
| **Legal/Security** | 日次+週次 | ツール数式、添付文書、npm audit | 要修正リスト、法改正アラート |
| **CS** | リアルタイム+週次 | 問合せ、X言及 | 返信ドラフト、フィードバック分類 |

### 共有コンテキスト層

- docs/（SSOT: Single Source of Truth）
- メトリクスDB（GA4 + KV + Supabase）
- git history
- ユーザーフィードバックキュー

### 段階的導入

| Phase | 内容 | 自動化レベル |
|-------|------|------------|
| Phase 1（今すぐ） | レポート生成のみ。Claude会話/Claude Codeで手動実行 | 情報収集 |
| Phase 2（インフラ後） | ドラフト作成。Workers + cron + Claude API | ドラフト → 人間承認 |
| Phase 3（信頼性確認後） | 自動実行 + 承認ゲート。低リスク=全自動、高リスク=承認必須 | 自律運用 |

### 承認ゲート（Human-in-the-loop）

自動実行不可: コードデプロイ、価格変更、法務判断、利用規約変更
自動実行可: X投稿、メトリクス収集、競合監視、SEO順位追跡

### CEOコクピットダッシュボード（/admin/cockpit/ — モバイル最適化）

全エージェントの出力をスマホ1画面でリアルタイム監視・承認するための管理画面。
外出先でもスマホ1つで会社全体を運用できる状態を目指す。

**ダッシュボード構成:**

| セクション | 表示内容 | データソース |
|-----------|---------|------------|
| **ヘルスサマリー** | 今日のPV/DAU/エラー率/収益のスパークライン | Analytics Agent |
| **承認キュー** | 承認待ちアクション一覧（スワイプで承認/却下） | 全エージェント |
| **アラート** | 異常検知・セキュリティ・法務リスク（赤/黄/緑） | Legal + Analytics |
| **SNSフィード** | X投稿ドラフト + エゴサ結果 + 返信ドラフト | Marketing + CS |
| **競合ウォッチ** | 直近の競合変更・新規参入アラート | Research |
| **週次レポート** | CEO Orchestratorの週次分析サマリー | CEO Orchestrator |

**技術スタック:**
- フロント: /admin/cockpit/ にReact SPA（PWA対応、ホーム画面追加可能）
- バック: Cloudflare Worker API（既存admin認証を流用）
- データ: KV（Phase 1）→ Supabase（Phase 2）にエージェント出力を蓄積
- 通知: 高優先アラートはWebPush + メール

**承認UX:**
- 各承認アイテムにコンテキスト（なぜこのアクションが必要か）+ プレビュー
- スワイプ右=承認、スワイプ左=却下、タップ=詳細展開
- 承認後は自動実行（X投稿→ポスト、コード→PR作成、返信→送信）

**段階的機能拡張:**
- Phase 1: メトリクス表示 + 手動レポート閲覧（既存/admin/の拡張）
- Phase 2: 承認キュー + アラート + SNSフィード
- Phase 3: 全エージェント統合 + ワンタップ承認 + 自動エスカレーション
