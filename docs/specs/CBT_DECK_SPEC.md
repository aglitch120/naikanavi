# CBTデッキ構築仕様書

## 概要

CATO令和7年度版CBT出題基準（98ページPDF）を構造化データにパースし、
全学修目標をフラッシュカード化する。国試デッキとは**完全独立**。

---

## Phase 1: 出題基準PDFパース

### ソース
`https://www.cato.or.jp/pdf/cbt_igakuR7.pdf`

### パース対象（p.6〜p.98）

出題基準の全項目。前文（p.1-5）はスキップ。

### データ構造

```typescript
interface CbtObjective {
  code: string;           // "C-2-5)-②" 等
  section: string;        // 大分類 "A" | "B" | "C" | "D" | "E" | "F"
  subsection: string;     // "C-2-5) 生体物質の代謝"
  topic: string;          // "C-2-5)-(1) 糖代謝" 等（あれば）
  objective: string;      // 学修目標テキスト全文
  depth: "説明" | "概説" | "列挙" | "図示";  // 末尾動詞から抽出
  diseaseClass?: "Ⅰ" | "Ⅱ";  // D領域の疾患分類（あれば）
}
```

|---------|------|-----------|

### 注意事項
- `＜削除＞` 記載のある項目はスキップ（例：A-2, A-3, A-4, A-9, B-2-2）
- 腫瘍は C-4-6 から E-3 に統合されている
- D領域の症候は F-1 にまとめられている

## Phase 3: カード生成仕様

### 通常カード（デッキ1-14）

国試デッキと同じJSON構造。1つの学修目標 → 1〜3枚のカード。

```json
{
  "id": "cbt-biochemistry-001",
  "front": "解糖系の律速酵素を3つ挙げ、各段階での調節機構を説明せよ",
  "back": "①ヘキソキナーゼ（G6Pで阻害）②ホスホフルクトキナーゼ-1（ATP阻害、AMP・F2,6BPで活性化）③ピルビン酸キナーゼ（ATP・アラニンで阻害）",
  "source_code": "C-2-5)-②",
  "deck": "cbt-biochemistry",
  "tags": ["代謝", "解糖", "律速酵素"]
}
```

### カード生成ルール

1. **depth="説明"の項目** → 詳細カード（具体的メカニズム・病態まで問う）
2. **depth="概説"の項目** → 概要カード（定義・分類・列挙レベル）
3. **depth="列挙"の項目** → リストカード（列挙+1行解説）
4. **1学修目標あたり1-3枚**（情報量に応じて分割）
5. **各カードにsource_code紐付け** → 網羅率100%を証明可能

### 4段階ステップカード（デッキ15: 臨床推論）

CBT独自の順次解答型4連問に対応する特殊フォーマット。

### ステップカード生成ルール
- **Ⅰ群疾患のみ**を対象（4連問で出題される疾患群）
- 各症例4ステップ固定: 医療面接→身体診察→検査→病態生理
- Step4は必ず基礎医学的要素を含める（CBTの特徴）
- 40-50症例で主要Ⅰ群疾患をカバー

## Phase 4: 実装手順（Claude Code用）

### Step 1: PDFダウンロード＆パース
```bash
# PDFをダウンロード
curl -o cbt_r7.pdf https://www.cato.or.jp/pdf/cbt_igakuR7.pdf

# Pythonでパース（pdfplumber推奨）
# → cbt_objectives.json に全学修目標を出力
```

### Step 2: 構造化データ検証
- 全項目数をカウント（推定500-600項目）
- セクション別の項目数を集計
- 削除セクションが除外されていることを確認

### Step 3: カード生成
- cbt_objectives.json → 各デッキのカードJSONを生成
- 1学修目標 → 1-3カード（AI生成）
- source_codeを全カードに付与

### Step 4: ステップカード生成
- Ⅰ群疾患リストを抽出
- 各疾患に対して4段階ステップカードを生成
- 40-50症例

### Step 5: デッキファイル配置
- 既存の国試デッキと同じディレクトリ構造で配置
- `public/data/decks/cbt-*.json`

### Step 6: UI対応
- デッキ選択画面に「CBT対策」カテゴリを追加
- ステップカードの表示UI（4段階フリップ）を実装

## 品質基準

1. **網羅率100%**: 出題基準の全学修目標にsource_codeが紐付いたカードが存在すること
2. **正確性**: 医学的に正確な内容（web検索で最新ガイドラインを確認）
3. **CBT深度**: 「概説」レベルの項目に国試レベルの詳細を書かない
4. **独立性**: 国試デッキを参照しなくてもCBTデッキだけで完結すること
