### 日本医療IT SEOの特性
- 検索ボリュームは小さいが転換率が極めて高い（「J-OSLER 病歴要約 書き方」で検索する人＝100%専攻医）
- 競合の多くは個人ブロガー→体系的SEOをやっているプレイヤーがいない
- 長尾KWが英語圏より重要（日本語は助詞+漢字で組み合わせが多い）

## 3. 資本ある競合のSEO戦略分析

### HOKUTO (hokuto.app) のSEO戦略

**現状**: アプリファーストだったが、2024-2025年にWeb版を本格展開。SEO強化中。
**Webサイト構造** (スクレイピング結果):
- `/calculator/` — 500+計算ツール（各ツールが独立ページ: eGFR, CHA2DS2-VASc等）
- `/article/` — 医療記事/ニュース
- `/regimen/` — 抗がん剤レジメン情報
- `/medicine/` — 薬剤情報DB
- `/guide/` — 抗菌薬ガイド/ERマニュアル

**SEO強み**:
1. Programmatic SEO: 500+計算ツールページが自動生成→「eGFR 計算」等の検索でHOKUTOが上位
2. 医学的権威性: 監修医師プロフィールを各ページに表示（E-E-A-T）
3. 構造化データ: JSON-LD実装済み
4. 大量の薬剤/GL情報ページ

**SEO弱み**:
1. J-OSLER関連コンテンツ: **ゼロ**（iworの独占領域）
2. Study/暗記カード: **ゼロ**
3. マッチング/キャリア: **ゼロ**（レジナビはレビューサイトのみ）
4. ブログ/教育コンテンツ: 薄い（計算ツール+薬剤DBに特化）
5. 個人の体験談/ストーリー: なし（プロダクトページのみ）

**iworの対HOKUTO戦略**:
- 計算ツールでは正面から競わない（HOKUTO 500+ vs iwor 166）
- J-OSLER/専門医試験/マッチング/暗記カードで圧倒する
- SEO記事×ツール連携で「記事を読んだらツールを使う」体験を作る
- HOKUTOにない「教育コンテンツ」で勝つ

### m3.com のSEO戦略

**現状**: 日本の医師の90%以上が登録。巨大なドメインパワー。
**Webサイト構造**:
- `/open/` — ログイン不要の公開ページ（SEO用）
- 医療ニュース / 求人 / 学会情報 / ポイント
- **重要**: m3.comの多くのコンテンツはログイン必須→SEO効果が限定的

**SEO強み**:
1. 圧倒的ドメインパワー（DR 70+）
2. 医療ニュースの速報性
3. 求人ページの大量インデックス

**SEO弱み**:
1. ログイン壁: 主要コンテンツがログイン後→Googlebotがクロールできない
2. UX: 古いデザイン、広告過多→直帰率高い
3. 教育コンテンツ: ニュース中心で「HowTo」型記事が少ない
4. 計算ツール: なし
5. J-OSLER/試験対策: なし

**iworの対m3戦略**:
- m3.comがカバーしない「HowTo」領域で勝つ
- 全コンテンツをオープンWebで公開（ログイン壁なし）→SEO圧倒的有利
- m3.comのドメインパワーには勝てないが、ニッチKW×深さで上回る

## 4. トピッククラスター（11大テーマ）

### 設計原則
- **ペルソナ別ではなくテーマ別**: 「バイト」は専攻医も一般医師も検索する
- **各テーマに1つのピラーページ**（3,000-5,000字の包括ガイド）
- **ピラーから各記事へ、各記事からピラーへ相互リンク**
- **1記事1キーワード原則**: カニバリ防止
- **既存173記事を全て新クラスターにマッピング**

### CSVテーマ番号 → 11クラスター マッピング

CSVの細分テーマ番号（分析用の粒度）を、11クラスターに対応付ける。
CSVは書き換えず、このマッピングでブログ記事のクラスター帰属を決定する。

| クラスター | CSVテーマ番号 | 備考 |

| **1. J-OSLER** | `1-josler`, `14-app-josler` | |
| **2. 専門医試験** | `2-exam` | |
