## 2. 技術スタック

```
フロント:  Next.js 14 + MDX + Tailwind CSS → Cloudflare Pages
API:       Cloudflare Workers
KV:        Cloudflare KV（キャッシュ・セッション）
認証+DB:   Supabase（Auth + PostgreSQL）
論文自動化: n8n + PubMed API + Claude API
決済:      BOOTH（Phase 1）→ Stripe（Phase 2）
```

### Supabase選定理由
- Auth組み込み（メール認証、OAuth、セッション管理）
- PostgreSQL（業界標準、売却時に買い手が安心）
- Row Level Security（ユーザー間のデータ分離）
- リアルタイム（病棟ダッシュボードのTODO即時反映）
- 無料枠でPhase 1は余裕（500MB, 50K MAU）

## 4. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| PRO課金されない | 収益ゼロ | まずJ-OSLERテンプレ単品で検証 |
| SEOアルゴリズム変動 | PV半減 | メール配信・SNSで直接接点確保 |
| J-OSLER制度変更 | コア商品の価値消失 | 臨床ツール＋病棟ダッシュ（制度非依存）を早期構築 |
| 医療情報の誤り | 炎上・信頼失墜 | 免責表示、出典明記、誤り報告フォーム |
| 勤務先バレ | 副業禁止の場合 | Phase 1はBOOTH匿名、Phase 2で法人化 |
| Supabase障害 | PRO機能全停止 | 定期バックアップ、Cloudflare KVにキャッシュ |
